01The mental model: formatting is a contract

Functions such as printf and scanf use a format string as a compact description of how data should be converted. The format string and the arguments must agree about types. When they do not agree, the result is not merely “a strange output”; in many cases the program has undefined behavior.

Output

printf takes values and converts them into text.

Value → format rule → characters

int age = 25;
printf("Age: %d\n", age);

Input

scanf reads text, converts it, and stores the result through pointers.

Characters → format rule → object in memory

int age;
scanf("%d", &age);
Core distinction in printf, width and precision describe how output is displayed. In scanf, width describes the maximum number of input characters consumed. Similar syntax, different job.

02Anatomy of a printf conversion

The general shape is:

%flagswidth.precisionlengthspecifier

Only % and the final conversion specifier are normally required. Everything in between is optional.

printf("%+08d", 452);
        │││└─ d  : decimal integer
        ││└── 8  : minimum field width
        │└─── 0  : pad with zeroes
        └──── +  : always show the sign
+0000452

The scanf grammar is different

%*widthlengthspecifier

For scanf, * means “match this input but do not assign it.” There is no output-style +, -, #, or 0 flag system.

04printf flags — including interactions

Flags modify the presentation of a conversion. They appear after % and before width. The same set of flags is not valid for every conversion specifier.

FlagMeaningExampleTypical result
-Left-align inside the field.%-8d452     
+Always show a sign for a signed numeric conversion.%+d+452
spacePrefix a positive signed value with one blank.% d 452
0Pad a supported numeric field with leading zeroes.%08d00000452
#Request an alternative representation where the conversion defines one.%#x0x1c4

4.1 Flag order is flexible

The flags form a set. Their textual order does not change their meaning. These are equivalent:

printf("%+08d\n", 452);
printf("%0+8d\n", 452);
+0000452 +0000452

The same idea applies to %-+8d and %+-8d: both request an explicit sign and left alignment.

4.2 When - and 0 meet, - wins

printf(":%0-8d:\n", 452);
printf(":%-08d:\n", 452);
:452 : :452 :

Both GCC 14.2 and Clang 17 warn that the 0 flag is ignored when - is present. Left alignment needs padding on the right, so leading zero padding no longer makes sense.

4.3 When + and the space flag meet, + wins

printf(":%+ d:\n", 452);
printf(":% +d:\n", 452);
:+452: :+452:

The explicit + is stronger than the “blank for positive values” rule, so the space flag is ignored.

4.4 A flag can be valid for one conversion and invalid for another

Important distinction

%+20s, %020s, and %#i are not clever formatting tricks. They use flags for conversions where those flags have no defined meaning. Clang diagnoses these as format strings whose behavior is undefined. GCC also warns.

On the tested glibc runtime, %+20s happened to behave like %20s and %020s also happened to use spaces. That observation is useful for understanding what you may see during an experiment, but it is not a rule to teach or rely on.

05Field width and precision

5.1 Field width is a minimum, not a maximum

printf(":%4d:\n", 10);
printf(":%4d:\n", 45646546);
: 10: :45646546:

The field expands when the converted result needs more characters. Width by itself does not truncate numbers or strings.

5.2 Precision changes meaning with the conversion

ConversionMeaning of precisionExample
IntegerMinimum number of digits.%.6d → 000452
f, e, EDigits after the decimal point.%.2f → 4.49
g, GSignificant digits.%.4g
sMaximum bytes written from the string.%.5s → first five bytes

5.3 Integer precision overrides the 0 flag

printf(":%08.5d:\n", 452);
printf(":%8.5d:\n", 452);
: 00452: : 00452:

Once an explicit precision is given for an integer conversion, the 0 flag is ignored. Precision supplies the minimum digit count; ordinary field padding supplies the remaining spaces.

5.4 Precision zero has a special integer edge case

printf(":%.0d:\n", 0);
printf(":%#.0o:\n", 0u);
:: :0:

For an integer value of zero with precision zero, no digits are produced. Octal alternative form is the notable exception shown above: # still ensures a leading zero.

5.5 Dynamic width and precision with *

int width = 10;
int digits = 3;
double x = 12.34567;

printf("|%*.*f|\n", width, digits, x);
| 12.346|

The width and precision values are taken from preceding int arguments.

Negative dynamic width

printf("|%*d|\n", -8, 452);
|452 |

A negative width supplied through * means left alignment with the corresponding positive width.

Negative dynamic precision

printf("|%.*f|\n", -1, 12.34567);
|12.345670|

A negative precision supplied through * is treated as though the precision were omitted, so %f falls back to its default six digits after the decimal point.

6.3 Alternative form and zero

printf("%#x\n", 0u);
printf("%#08x\n", 452u);
printf("%#08o\n", 452u);
0 0x0001c4 00000704

Hexadecimal prefixes are not added for a zero value. With zero padding, the 0x prefix is placed before the padding zeroes.

%#i is not valid alternative-form output

The # flag has no defined meaning for %i. In the test run glibc happened to print the same decimal result, but Clang explicitly diagnosed the combination as undefined behavior. Use %#o, %#x, or %#X for base prefixes.

07Floating-point formatting in detail

First rule

In calls to printf, a float argument is promoted to double. Therefore %f and %lf produce the same output role in printf. In scanf, however, %f requires float * and %lf requires double *.

Using the exact values from the reference program:

float p = 4.49f;
float q = -0.00000005f;
float r = 100005.21454f;
float t = 78565465e-14f;

7.2 Why the stored value may surprise you

The literal 100005.21454 is converted to float when assigned to r. On the tested platform, the nearest stored value prints as 100005.2109375000 with enough decimal places. Formatting does not create that error; it merely reveals the approximation already stored.

7.3 Precision rounds the displayed result

printf("%.0f\n", p);
printf("%.1f\n", p);
printf("%.4f\n", p);
4 4.5 4.4900

Changing precision changes the text produced, not the object stored in memory.

7.4 Negative zero can appear

q is a very small negative value. When rounded to six decimal places it becomes -0.000000, and with zero fractional digits it becomes -0. The sign can survive even when the displayed magnitude rounds to zero.

7.5 The # flag can force the decimal point

printf("%.0f\n", 4.49);
printf("%#.0f\n", 4.49);
4 4.

7.6 - overrides zero padding here too

printf(":%-020.10f:\n", 0.0000007856546631);
:0.0000007857 :

GCC and Clang both warn that 0 is ignored because left alignment is active.

08String output formatting

char s[]   = "wow!";
char ss[]  = "wOw!Really Great!";
char sss[] = "100% accurAte! Handsome! Awesome! Speechless!";

8.1 Width right-aligns a short string

printf(":%20s:\n", s);
printf(":%20s:\n", ss);
printf(":%20s:\n", sss);
: wow!: : wOw!Really Great!: :100% accurAte! Handsome! Awesome! Speechless!:

Width is a minimum. The long third string is printed completely rather than being truncated.

8.2 Left alignment

printf(":%-20s:\n", s);
:wow! :

8.3 Precision limits how much of the string is written

printf(":%20.5s:\n", ss);
printf(":%-20.5s:\n", ss);
: wOw!R: :wOw!R :

Precision is applied first to obtain at most five bytes; width then aligns that shortened result inside a 20-character field.

8.4 Exact behavior of the experimental invalid flags

The reference code deliberately tries %020s, %+20s, and %-020s. Both GCC and Clang warn. Clang specifically says the unsupported flag combinations result in undefined behavior.

On this tested glibc run: %020s happened to look like %20s %+20s happened to look like %20s %-020s happened to look like %-20s
Do not convert an observation into a language rule

The runtime output above is only what this implementation happened to do. Portable C code should use %20s, %-20s, %20.5s, or %-20.5s.

09Character and simple string I/O

9.1 getchar() and putchar()

#include <ctype.h>
#include <stdio.h>

int main(void)
{
    int ch;

    while ((ch = getchar()) != EOF) {
        putchar(toupper((unsigned char) ch));
    }

    return 0;
}
Why int ch, not char ch?

getchar() needs to represent every possible unsigned-character value plus the separate sentinel EOF. Store its result in int.

9.2 putchar(c) and printf("%c", c)

Both can output one character. putchar is the direct character-output function; printf is useful when the character is part of a larger formatted record.

9.3 puts() appends a newline automatically

puts("-----------------------------");
printf("-----------------------------\n");

These produce the same visible line. The difference is that the newline is implicit in puts.

9.4 A character can also be formatted as an integer

char ch = 'g';
printf("character = %c, numeric code = %d\n", ch, (unsigned char) ch);
character = g, numeric code = 103

The value 103 is what the tested ASCII-based execution environment uses for 'g'. This explains the reference expression x += s[10]: before the array is overwritten, s[10] is 'g', so on this system x changes from 100 to 203.

10Formatted input with scanf

10.1 Field width and assignment suppression — exact reference example

Given:

63458465321654
int a, b, c, d;
int count = scanf("%1d%2d%*3d%4d%d", &a, &b, &c, &d);
ConversionCharacters consumedStored result
%1d6a = 6

The suppressed conversion consumes input but is not counted as an assignment.

10.2 Base-specific input: %o, %x, %X, and %d

With the exact reference input:

065355 0X4fFbFd 0xd 0123
unsigned int a, b, c;
int d;

sscanf("065355 0X4fFbFd 0xd 0123",
       "%o%x%X%d", &a, &b, &c, &d);
decimal values: 27373 5241853 13 123

The final token 0123 is read by %d, so it is decimal 123. The leading zero does not make %d switch to octal.

10.3 The same text with %i gives a different last value

int a, b, c, d;
sscanf("065355 0X4fFbFd 0xd 0123",
       "%i%i%i%i", &a, &b, &c, &d);
27373 5241853 13 83

%i detects the prefix: 065355 and 0123 are octal, while 0X4fFbFd and 0xd are hexadecimal.

10.4 Reformatting the parsed values

printf("a=%o, b=%x, c=%i, d=%X\n", a, b, (int)c, (unsigned)d);
printf("a=%#o, b=%#x, d=%#X\n", a, b, (unsigned)d);
a=65355, b=4ffbfd, c=13, d=7B a=065355, b=0x4ffbfd, d=0X7B

10.5 Correct pointer types are part of the format contract

Type mismatch

%o, %x, and %X expect unsigned int * unless a length modifier says otherwise. The original experiment used int *; that is a format-type mismatch even when int and unsigned int happen to have the same representation size.

10.6 Whitespace behavior is conversion-dependent

  • Most numeric conversions and %s skip leading whitespace automatically.
  • %c, %[, and %n do not automatically skip leading whitespace.
  • A whitespace character inside the format string means “skip any amount of input whitespace.”

10.7 Always check the return value

int x;
if (scanf("%d", &x) == 1) {
    printf("Read: %d\n", x);
} else {
    printf("Invalid integer input.\n");
}

scanf reports the number of successful assignments. Input failure before the first conversion may produce EOF.

11String pattern matching with scansets

A scanset uses %[...]. It reads a consecutive sequence of characters selected by the set and then appends a null terminator to the destination array.

11.1 Positive scanset

char s[100];
scanf("%99[a-z]", s);

This accepts up to 99 consecutive lowercase letters. Width 99 leaves one byte for '\0'.

11.2 Negated scanset

char s[100];
sscanf("sfsad RRRG;", "%99[^a-d]", s);
sfs

The leading ^ means “accept anything except the listed set.” Reading stops immediately before the first a, b, c, or d.

11.3 The exact chained scanset experiment

Starting with:

sfsad RRRG; - dkdfjalkds

The first conversion:

scanf("%[^a-d]", s);

stores sfs and leaves the first a unread. The next conversion:

scanf("%[-a-d; GR]", ss);

starts at that a. Because the leading - is treated as a literal hyphen in this scanset and a-d is a range, the accepted characters are the hyphen, a through d, semicolon, space, G, and R.

s = :sfs: ss = :ad RRRG; - d: remaining input begins with :kdfjalkds:

This exact result was confirmed under both tested runtimes.

11.4 Scansets do not skip leading whitespace automatically

If the next unread character is a space and space is not in the scanset, the conversion fails immediately. Add a space before the conversion in the format string when you intentionally want to skip input whitespace first:

scanf(" %99[a-z]", s);

11.5 Why %[] is malformed

Both GCC and Clang warn that "%[]" has no closing ] for a valid scanset specification. In the local experiment it returned zero assignments and left the destination unchanged, but malformed format specifications must not be used as a programming technique.

Practical rule

Always give string and scanset input a width that leaves room for the terminating null byte: %99s or %99[...] for a 100-byte array.

12Formatted I/O with strings: sscanf, sprintf, and snprintf

12.1 sscanf: parse text already stored in memory

char text[] = "ID=42 CGPA=3.75";
int id;
double cgpa;

int count = sscanf(text, "ID=%d CGPA=%lf", &id, &cgpa);
count = 2 id = 42 cgpa = 3.75

The source is a string instead of stdin, but conversion and return-value rules follow the formatted-input family.

12.2 Pointer arithmetic lets parsing begin in the middle

char s[] = "Something going to happen, but who knows!\0Oh!";
char word[100];

sscanf(s + 37, "%99s", word);
word = ows!

s + 37 points directly at the o in ows!.

12.3 sprintf: format into a character array

char buffer[100];
int count = sprintf(buffer, "x = %d", 203);
buffer = x = 203 count = 7

The return value is the number of characters written before the terminating null character. sprintf itself has no destination-size limit.

12.4 Prefer snprintf when capacity matters

char buffer[8];
int needed = snprintf(buffer, sizeof buffer, "value=%d", 12345);

At most seven visible characters plus the null terminator fit in buffer. The return value reports how many characters would have been produced with unlimited space, which lets you detect truncation by checking whether needed >= sizeof buffer.

13Embedded null characters, array capacity, and stale bytes

Consider the exact reference array:

char s[] = "Something going to happen, but who knows!\0Oh!";

There are two null bytes in the initialized content: the explicit \0 after the exclamation mark and the implicit terminator added after Oh!.

printf("sizeof = %zu\n", sizeof s);
printf("strlen = %zu\n", strlen(s));
printf("s      = %s\n", s);
sizeof = 46 strlen = 41 s = Something going to happen, but who knows!

sizeof measures the complete array. strlen counts only from the starting address up to the first null byte.

13.1 Critical indices before overwriting

IndexByteMeaning
10'g' / 103 on the tested ASCII systemUsed by x += s[10], making 100 become 203.
37'o's + 37 begins the substring ows!.
41'\0'First string terminator.
42–44O h !Bytes physically present in the array but hidden from %s starting at s.
45'\0'Final implicit terminator.

13.2 What sprintf overwrites — and what it does not

int x = 100;
x += s[10];              /* 203 on the tested ASCII system */
sprintf(s, "x = 100+s[10] = %d", x);
s = x = 100+s[10] = 203 strlen = 19

The new text occupies indices 0 through 18 and sprintf places its null terminator at index 19. It does not clear the rest of the 46-byte array.

printf("s+19 = :%s:\n", s + 19);
printf("s+20 = :%s:\n", s + 20);
s+19 = :: s+20 = :appen, but who knows!:

Index 19 is the new null terminator, so s + 19 is an empty string. Index 20 still contains old data beginning with the a from the original word happen.

Core model

An array is a region of bytes with a capacity. A C string is a view that starts at some byte and ends at the first null byte encountered from that starting point.

15A consolidated, corrected demonstration program

Verification environment

The edge-case programs behind this tutorial were compiled with gcc 14.2.0 and clang 17.0.0 using -std=c17 -Wall -Wextra -Wpedantic -Wformat=2. The two executables produced identical runtime output for the tested cases. Compiler warnings are treated as part of the lesson: a result that happens to print on one C library is not presented as guaranteed behavior when the format itself is invalid.

This program keeps the spirit of the reference code while making the type usage and bounded string operations suitable for teaching.

#include <ctype.h>
#include <stdio.h>
#include <string.h>

int main(void)
{
    /* Integer formatting */
    int a = 10, b = 452, c = -56465, d = 45646546;

    printf("Default : %d %d %d %d\n", a, b, c, d);
    printf("Width   : %8d %8d %8d %8d\n", a, b, c, d);
    printf("Zero    : %08d %08d %08d %08d\n", a, b, c, d);
    printf("Sign    : %+08d %+08d %+08d %+08d\n", a, b, c, d);
    printf("Left    : %-8d %-8d %-8d %-8d\n\n", a, b, c, d);

    /* Floating-point formatting */
    double p = 4.49;
    double q = -0.00000005;
    double r = 100005.21454;

    printf("Fixed       : %.6f\n", p);
    printf("Scientific  : %.6e\n", q);
    printf("Width/prec. : %20.10f\n", r);
    printf("Signed/zero : %+020.10f\n\n", p);

    /* String formatting */
    char text[] = "wOw!Really Great!";
    printf("Right   : :%20s:\n", text);
    printf("Left    : :%-20s:\n", text);
    printf("Limit   : :%20.5s:\n\n", text);

    /* sscanf: parse from a string */
    char record[] = "ID=42 CGPA=3.75";
    int id;
    double cgpa;

    if (sscanf(record, "ID=%d CGPA=%lf", &id, &cgpa) == 2) {
        printf("Parsed: id=%d, cgpa=%.2f\n", id, cgpa);
    }

    /* snprintf: format into a bounded array */
    char buffer[100];
    snprintf(buffer, sizeof buffer, "Student %d has CGPA %.2f", id, cgpa);
    printf("Buffer: %s\n", buffer);

    return 0;
}

scanf pattern

% [*] [maximum-width] [length] conversion
  • * suppresses assignment.
  • Width is the maximum number of input characters for that conversion.
  • %i auto-detects common integer base prefixes.
  • %[...] performs scanset matching.
  • Except for arrays already decaying to pointers, destinations are normally passed with &.
  • Always inspect the return value when input validity matters.

Recommended teaching order start with specifiers → width → precision → flags → integer examples → floating-point examples → strings → scanf width/suppression → scansets → sscanf/snprintf → null-character memory behavior.

Formatting becomes much easier once you read a conversion specification as a small grammar: flags control style, width controls the field, precision controls detail, and the final conversion determines how the value is interpreted.
