01The array mental model

An array is one named block containing a fixed number of elements of the same type. The elements are distinguished by indexes. Indexing begins at 0, not 1.

int marks[5] = {72, 85, 91, 68, 77};

printf("%d\n", marks[0]);  /* first element: 72 */
printf("%d\n", marks[4]);  /* last element: 77  */
For N elements, valid indexes are 0 through N - 1.
Array boundary

For marks[5], the last valid element is marks[4]. Accessing marks[5], marks[-1], or any other outside position is invalid and causes undefined behavior.

02Array declaration and initialization

DeclarationWhat it creates
int a[5];Five local integer elements. Their initial values are unknown until the program stores values in them.
int a[5] = {1, 2, 3, 4, 5};Five elements with all initial values supplied.
int a[] = {1, 2, 3};The compiler counts the values and creates three elements.
int a[5] = {1, 2};The result is {1, 2, 0, 0, 0}. Missing initializers become zero.
int a[5] = {0};All five elements become zero.
Initialization is part of declaration

After the declaration, change individual elements: a[0] = 10;. C does not allow a = {1, 2, 3}; later, and one whole array cannot be assigned to another with a = b;.

03How an array is stored in memory

Every array element is stored immediately after the previous element. This is called contiguous or consecutive storage.

ar[0]101000
ar[1]201004
ar[2]301008
ar[3]401012
ar[4]501016

Example only: the first address is 1000 and each int uses 4 bytes. Actual addresses vary.

address of ar[i] = starting address + i × size of one element

If an int occupies 4 bytes, neighboring integer elements begin 4 bytes apart. If a char occupies 1 byte, neighboring character elements begin 1 byte apart. There is still no unused gap between elements.

04Why the array name represents the start

In many expressions, C uses an array name such as ar to represent the location where that array begins. The array begins with its first element, ar[0].

ar → starting address of the array → address of ar[0]

This is why a function that needs the array's starting storage can receive the array name. For this foundation lesson, that memory picture is enough; the exact formal rule belongs later in the course.

05Why ar + 1 reaches the next element

The + 1 means “move forward by one element.” It does not mean “add one byte.” C already knows the type and size of each array element, so it calculates the required byte jump.

ExpressionLocation representedExample address
ar or ar + 0ar[0]1000
ar + 1ar[1]1004
ar + 2ar[2]1008
The wording matters

ar + 1 does produce another memory address. The important point is that it reaches the next element's address, not merely the next byte address.

06Traversing an array

Traversal means visiting every element systematically. For five elements, the loop visits indexes 0, 1, 2, 3, 4.

#include <stdio.h>

int main(void) {
    int numbers[5];

    for (int i = 0; i < 5; i++) {
        scanf("%d", &numbers[i]);
    }

    for (int i = 0; i < 5; i++) {
        printf("%d ", numbers[i]);
    }
    printf("\n");
    return 0;
}
Input detail

numbers[i] is one ordinary integer element. Therefore scanf receives its address as &numbers[i], just as it would receive &age for one integer variable.

07Character array versus string

A character array is simply an array whose elements are char. A C string is a character sequence whose end is marked by '\0'. Therefore:

Every writable C string can live in a char array, but not every char array is a string.

Character array, not a string

char a[3] = {'C', 'A', 'T'};

All three positions contain visible characters. There is no '\0' to mark the end.

Character array and string

char s[4] = {'C', 'A', 'T', '\0'};

The fourth position marks the end, so the stored string is "CAT".

Character constant versus string literal

'A' is one character value. It can be stored in one char element.

"A" is a string literal containing two character positions: 'A' followed by '\0'.

Why this distinction matters

printf("%s", a) requires a string. If the array contains no '\0', output may continue beyond the array while searching for an end marker. That is undefined behavior.

08String declaration and initialization

DeclarationActual stored elements
char s[] = "Hello";Six elements: 'H','e','l','l','o','\0'. The compiler determines size 6.
char s[6] = "Hello";Exactly enough space for five visible characters and the terminator.
char s[10] = "Hello";The first six positions hold the string; the remaining four positions are also zero-initialized.
char s[] = {'H','i','\0'};A manually initialized valid string of length 2.
char s[] = {'H','i'};A two-element character array, not a string.
No room for the terminator

char s[5] = "Hello"; has room for the five letters but not '\0'. Some compilers warn. The result must not be treated as a string.

09Why string input does not use &

For one integer, scanf needs the variable's storage address:

int age;
scanf("%d", &age);

For a character array, the array name already represents its starting address:

char name[20];
scanf("%19s", name);
One rule, two forms

&age is needed because age is one variable. For the array, name already supplies the starting location, so do not write &name.

Why the width is 19

The array has 20 positions. %19s reads at most 19 non-whitespace characters and then adds '\0' in the final required position. Plain %s can overflow the array if the word is too long.

%s stops at whitespace, so it reads one word rather than a complete name containing spaces.

10The null character

The null character is written '\0' and has numeric value zero. It occupies one character position in memory but produces no visible output. String operations treat the first null character as the end.

text[0]'C'2000
text[1]'A'2001
text[2]'T'2002
text[3]'\0'2003

Illustrative addresses show the characters stored consecutively. The null character occupies the next character position after 'T'.

NotationMeaning
'\0'One null character with value zero.
'0'The visible digit character zero. It is not the null character.
0An integer zero. Storing it in a char element creates a null character.
"\0"A string containing an explicit null plus the automatic terminator, so it contains two null characters.

11Putting '\0' in the middle

The first null becomes the logical end of the string. Later array elements still exist and can be accessed by index, but string output stops before them.

#include <stdio.h>

int main(void) {
    char text[6] = {'H', 'i', '\0', 'X', 'Y', '\0'};

    printf("string: %s\n", text);
    printf("text[3]: %c\n", text[3]);
    return 0;
}
string: Hi text[3]: X

The array contains six character positions. As a string, however, it represents only "Hi".

12String traversal and length

An ordinary array loop stops at a known element count. A string loop normally stops when it finds '\0'.

Print one character at a time

char text[] = "Bangladesh";

for (int i = 0; text[i] != '\0'; i++) {
    printf("%c\n", text[i]);
}

Count the length manually

char text[] = "Bangladesh";
int length = 0;

while (text[length] != '\0') {
    length++;
}

printf("Length = %d\n", length);
Length = 10
What length counts

Length counts the characters before the first '\0'. It does not count the null character itself.

13Size, capacity, and length

char text[10] = "cat";
IdeaMeaningResult here
Array size/capacityTotal character positions reserved by the declaration.10 positions
String lengthCharacters before the first '\0'.3 characters
Storage currently neededLength plus one position for '\0'.4 positions
sizeof text is 10 · calculated length is 3 · length + terminator is 4

For char exact[] = "cat";, the compiler reserves exactly four positions: three letters and one null character. Therefore the array size is 4 while the string length is 3.

Integer-array size

For int ar[5], the total byte size is 5 × sizeof ar[0]. The expression sizeof ar / sizeof ar[0] gives the element count where the array is declared.

14Common mistakes and their causes

Starting at index 1

This skips the first element. Array traversal normally begins at 0.

Using i <= 5

For five elements, this reaches invalid index 5. Use i < 5.

Printing a non-string with %s

Without '\0', output has no valid stopping marker.

Using &name with %s

The array name already gives the starting location. Use name.

Confusing '\0' and '0'

'\0' ends a string; '0' is a visible digit.

Confusing size and length

Size is reserved storage. Length stops at the first null character.

15Practice and quick checks

  1. Declare an integer array of six elements, initialize only the first two, and predict every stored value.
  2. Read five integers and print them in forward and reverse order.
  3. For an int array beginning at address 2000 with four-byte elements, calculate the example addresses of indexes 0 through 4.
  4. Explain in one sentence why ar + 1 does not mean “the next byte.”
  5. Write one character array that is not a string and one that is a valid string.
  6. Predict the size and length of char word[12] = "array";.
  7. Put '\0' at word[2], then predict the output and recalculated length.
  8. Write a loop that counts vowels until it reaches the null character.
Final mental model

An array is a fixed, consecutive block of same-type elements. A string is a character sequence inside such storage whose logical end is the first '\0'. Indexes select elements, the array name identifies the start, and length is not the same as reserved size.
